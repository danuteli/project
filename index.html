Semantic HTML

Accessibility | SEO | Easy to understand

<header> represents introductory content, typically a group of introductory or navigational aids (also logo, search form, author name etc)
<nav> represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents
<article> represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable (e.g., in syndication)
<figure> represents self-contained content, potentially with an optional caption, which is specified using the (<figcaption>) element
<footer> contains information about the author of the section, copyright data or links to related documents
<main> separate from the <footer> and <nav>
<section> content with the same theme
<aside> additional information next to a main piece of content (sidebar)
<figure> encapsulates media (image, illustration etc)
<audio>
<video>
<embed> alternative for video, audio, img
